Introduction

Main Goal

A system that:

  1. Takes a bare C/C++ project as input.
  2. Gathers source code and info about the project.
  3. Calls an LLM to create a new fuzzing harness [1] for the project.
  4. Builds harness, executes it and evaluates it.
  5. If the harness passes the evaluation, integrates it into project (TODO).

Related work

  1. IRIS [2]: Neurosymbolic system using LLMs for static analysis.
  2. ProphetFuzz [3]: LLM-assisted fuzzing for option combinations.
  3. FuzzGPT & TitanFuzz [4], [5]: LLM-generated Python DL programs, used as inputs for fuzzing DL frameworks, e.g. PyTorch and TensorFlow [6], [7].
  4. KLEE [8]: Symbolic execution tool designed to automatically generate high-coverage test cases.

KLEE: 2008

Related work

Google

  1. OSS-Fuzz-Gen [9]: Automatic generation of harnesses for OSS-Fuzz projects [10]. Preexisting harnesses needed.
  2. FUDGE [11]: Closed-source program for automatic harness generation of C/C++ projects through LLMs.
  3. FuzzGen [12]: Automatic harness generation through whole system analysis. Uses programs using the API under test to generate a dependency graph of the API functions.

Libraries Used

DSPy

Declarative Self-improving Python

Declarative Python framework for programming, instead of prompting, LLMs [13]. Developed by Stanford’s NLP research team.

DSPy

Modules

DSPy provides built-in modules to generically compose LLMs and prompting techniques, e.g. modules corresponding to “Chain of Thought” [14] and “ReAct” [15] prompting techniques.

import dspy
lm = dspy.LM('openai/gpt-4o-mini', api_key='YOUR_OPENAI_API_KEY')
dspy.configure(lm=lm)

math = dspy.ChainOfThought("question -> answer: float")
math(question="Two dice are tossed. What is the probability that the sum equals two?")

DSPy

Prompt Optimizing

Given a few tens or hundreds of representative inputs of your task and a metric that can measure the quality of your system’s outputs, you can use a DSPy optimizer.

Development Tools

Development Tools

  • uv: Python package and project manager (poetry replacement).
  • ruff: Linter & formatter.
  • mypy: Static type checker.
  • pre-commit: Declarative manager of pre-commit git hooks.
  • pytest: Python test framework.
  • pdoc: Automatic API documentation.

Current Status

Features

  • Works for simple projects.
  • All source code files are included in the prompt.
  • Simple query to LLM (no prompting techniques yet).
  • Builds the harness semi-automatically.
  • Simple evaluation:
    1. Built succesfully?
    2. Times out in a reasonable timeframe?
    3. Produced at least one new crash file?
